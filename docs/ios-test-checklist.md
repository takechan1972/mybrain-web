# AIプラライト iPhone 15 Pro 実機テスト チェックリスト

対象：EAS Build の **preview-dev**（プラン切替可）/ **preview**（プランロック）で作成した iOS internal ビルド。
Expo Go はSDK不一致のため使用不可。EAS Build の internal 配布を実機へインストールして確認する。

---

## 0. 事前準備（Apple Developer 関連）
- [ ] Apple Developer Program（有料・年額）に登録済み
- [ ] `eas device:create` でテスト端末（iPhone 15 Pro）の UDID を登録した
- [ ] ビルド時に Apple ID でログインし、Provisioning Profile に端末が含まれている
- [ ] iPhone 側で **設定 → プライバシーとセキュリティ → デベロッパモード** を有効化（再起動が必要な場合あり）
- [ ] インストール後、**設定 → 一般 → VPNとデバイス管理** で開発元を信頼

## 1. 起動・外観
- [ ] 初回起動できる（クラッシュしない）
- [ ] スプラッシュがブランド青(#208AEF)＋アイコンで表示（Expo既定ロゴが出ない）
- [ ] ホーム画面アイコンが「AIプラ」アイコン
- [ ] アプリ名が「AIプラ」

## 2. 設定・接続
- [ ] 設定画面が開ける／「はじめての使い方」表示
- [ ] 「利用状況」カードを開ける（今日のチャット/音声入力、今月の文字起こし/予約自動登録）
- [ ] 「接続を確認」で接続状態が「確認済み」になる（テストAPI `https://aipla-api-test.onrender.com`）
- [ ] 通常画面に localhost / mock / dev / debug / undefined / null が出ない

## 3. メモ
- [ ] メモ新規作成／編集／削除（削除は確認ダイアログ）
- [ ] メモ音声入力（タイトル／本文／まとめて）
- [ ] **初回の音声入力時にマイク／音声認識の権限ダイアログが日本語で表示される**
- [ ] 音声入力後に手修正できる

## 4. 予定
- [ ] 予定新規作成／編集／削除（削除は確認ダイアログ）
- [ ] 予定タイトル音声入力／予定メモ音声入力
- [ ] 音声で日時入力（「明日の午後3時に打ち合わせ」→ 日時欄に整形される）
- [ ] 日時判定不可時「日時を自動判定できませんでした。手動で選択してください。」

## 5. AIチャット・プラン制限（preview-dev でプランを切替えて確認）
- [ ] AIチャット送信できる
- [ ] **Free：AIチャットが3回まで、4回目で「本日の利用上限に達しました。…」**
- [ ] **Free：メモ音声入力が3回まで、4回目で「本日の音声入力回数に達しました。…」**
- [ ] **Free：予定音声入力が3回まで**
- [ ] Free：文字起こし・予約自動登録がブロックされる
- [ ] **上限到達後も、手入力でメモ・予定を保存できる**
- [ ] preview-dev でプランを basic/standard/pro に切替えると上限値が変わる
- [ ] **preview（販売候補）ではプラン変更UIが出ず、現在プランが読み取り専用表示**

## 6. 契約・サポート
- [ ] お問い合わせ送信→受付番号・送信日時が表示
- [ ] 解約申請送信→受付番号表示（即時解約・即時削除されない）
- [ ] 規約／プライバシー／特商法リンク（未設定時は「準備中」）

## 7. データ保持・安定性
- [ ] アプリ完全終了→再起動後もメモ／予定／利用カウントが保持
- [ ] 権限を「許可しない」にしてもアプリが落ちず、手入力に戻れる
- [ ] 画面・ログに APIキー／token／本文全文など秘密情報が出ない

## EAS ビルドコマンド（iOS）
```bash
# 端末登録（初回のみ／未登録の端末を追加するとき）
eas device:create

# プラン切替テスト用（development挙動・プラン変更可）
eas build --platform ios --profile preview-dev

# 販売候補確認用（production挙動・プランロック）
eas build --platform ios --profile preview
```

## Apple Developer 関連の注意
- iOS の internal 配布には **Apple Developer Program（有料）** が必須。未登録だと `eas build --platform ios` の資格情報設定の段階で止まる。
- `eas device:create` で iPhone の UDID を登録しないと、ビルドはできてもインストール（信頼）できない。
- iOS 16+ は **デベロッパモード** を有効化しないと開発ビルドが起動しない。
- Android（APK）は Apple Developer 不要で手軽に確認できるため、先に Android preview-dev で挙動を確認しておくと効率的。
