## 1. 目的

既存の Drive 参照メモ機能は「読み込み → 検索の参考 → AI 相談の参考」まで動作している（OBS29R／OBS31R で本番QA完了）。ただし、**いま何件・どのメモが読み込まれているか**、**AI に実際に渡るのはどれか**、**どうやって外すか**が分かりにくい。
本メモは、この機能の**表示（UX）だけ**を対象に、小さな MVP 改善案を設計する。**データの持ち方・保存方針・スコープ・API・取り込み（インポート）・双方向同期は一切変更しない。**

---

## 2. 現状の実装（調査結果）

### 2.1 データと上限
- 参照メモの実体：`components/DesktopMemos.tsx` の `driveRefMemos`（`useState<DriveReferenceMemo[]>`）。**React state のみ・保存しない**（Supabase にも localStorage にも入れない）。リロードで消える。
- 追加：`DriveExportedFilesList` の「参照に追加」→ 親の `setDriveRefMemos` を呼ぶ。`fileId` で重複判定（重複追加はしない・案内を出す）。
- 削除：`removeDriveRef(fileId)`（1件）／`clearDriveRefs()`（全消し）。どちらもメモリのみ。
- AI へ渡す上限（`DesktopMemos.tsx`）：`DRIVE_REF_AI_MAX_ITEMS = 5`（先頭5件）・`DRIVE_REF_AI_MAX_CHARS = 200`（本文/件）・`DRIVE_REF_AI_MAX_TITLE = 60`。`buildDriveReferenceBlock()` が先頭5件を整形して user メッセージ末尾に別ブロックで足す。参照0件なら空文字＝**送信内容は従来と完全に同じ**。

### 2.2 いま参照メモが「見える」場所（現状の弱点）
1. **Drive 一覧の「参照中」バッジ**（`DriveExportedFilesList`）。ただし表示には「一覧を確認」で Drive を再取得する必要があり、**一覧を開いていないと見えない**。しかもバッジのみで、ここから外せない。
2. **検索結果ブロック**（`DesktopMemos.tsx`：`mode === 'list' && driveRefMemos.length > 0 && query.trim().length > 0`）。ここに「すべて解除」と各件「参照を解除」があるが、**検索語を入力しているときしか表示されない**。
3. **AI 通知**（AI アシスタントパネル内・`driveRefMemos.length > 0` のとき）：「Google Drive参照 N件も参考にします（MyBrainには保存されません）」。件数は `Math.min(len, 5)`。ただし**メモを選択して AI パネルを開いているときだけ**表示され、**どのメモが渡るかは示さない**。

## 3. UX ペインポイント（今回つぶしたいもの）

- **P1：読み込み済みの一覧が常時見えない。** 検索語を打たない限り、いま何件・どのメモが読み込まれているか分からない（検索結果ブロック依存）。
- **P2：1件だけ外す導線が検索語依存。** 「参照を解除」は検索結果ブロックの中だけ。検索していない状態では個別に外せない（全消しも同様に検索語依存）。
- **P3：一時的・保存されないことが常時見えない。** 「リロードで消える」警告は検索結果ブロックと AI 通知の中だけ。読み込んだ直後や検索前には出ない。
- **P4：AI に渡るのがどれか分からない。** 6件以上読み込むと先頭5件だけ渡るが、**どの5件か**が画面に出ない（読み込み順の先頭）。各200字までの切り詰めも見えない。
- **P5：上限（最大5件・約200字）が目立たない。** 数値がUIにほとんど現れない。
- **P6：フロー（プレビュー→参照追加→検索／AI）が分散して分かりにくい。** 追加した後、次に何を見ればよいかの手がかりが弱い。

---

## 4. MVP UX 改善案（小さく・表示のみ）

中心は **「読み込み済み Google Drive 参照メモ」トレイ**を1つ用意し、参照が1件でもあれば**検索語の有無に関係なく常時表示**すること。既存の検索結果ブロック・AI通知・Drive一覧はそのまま残す（役割の重複は許容し、まず「常に見える置き場」を作る）。

### 4.1 読み込み済み参照メモ・トレイ（新規・表示のみ）
- **表示条件**：`driveRefMemos.length > 0`（`mode === 'list'`）。検索語は不要。
- **置き場所（案）**：メモ一覧カラムの上部、既存の検索結果ブロックより前。`DesktopMemos.tsx` 内に閉じたコンポーネント（新 state を増やさない／`driveRefMemos`・`removeDriveRef`・`clearDriveRefs` を使うだけ）。
- **中身**：
  - 見出し＋件数：「読み込み中の Google Drive 参照メモ（N件）」。
  - 常時の一時性警告（P3）：「これらは参考用です。MyBrain には保存されません。ページを再読み込みすると消えます。」
  - 上限の明示（P5）：「AI 相談で使えるのは先頭5件・各メモ約200字までです。」
  - 各件行：タイトル（無ければファイル名）＋ファイル名＋「参照を解除」ボタン（`removeDriveRef(fileId)`）。→ P1・P2 を解決。
  - 「すべて解除」ボタン（`clearDriveRefs()`）を見出し横に。→ P2 を解決。
  - **AI 対象の可視化（P4）**：先頭5件には小さなラベル「AIで使用」、6件目以降には「AIには渡りません（6件目以降）」を付ける。並びは `driveRefMemos` の読み込み順（＝ `buildDriveReferenceBlock` の先頭5件と一致）。ロジックは変えず、**表示で `index < 5` を判定するだけ**。

### 4.2 AI 相談前の「使う参照」表示の強化（表示のみ）
- 既存の AI 通知（`driveRefMemos.length > 0` のとき表示）を、件数だけでなく**実際に渡る先頭5件のタイトル**を軽く示す形に強化する。
  - 例：「Google Drive参照 3件も参考にします（MyBrainには保存されません）」＋その下に先頭最大5件のタイトルを小さく列挙（`driveRefMemos.slice(0, 5)` を表示するだけ・送信ロジックは不変）。
- **送信内容は変えない。** 表示に使う件数・タイトルは既存の `DRIVE_REF_AI_MAX_ITEMS`／`buildDriveReferenceBlock` と同じ基準を参照するだけ。

### 4.3 導線のわかりやすさ（軽微・文言のみ）
- `DriveExportedFilesList` の「参照に追加」直後の案内（`addNote`）に、次の一歩を一言添える：「参照に追加しました。読み込み中の一覧は左のメモ一覧上部に表示されます。」（文言のみ・新ロジックなし）。

### 4.4 やさしい日本語の文言（非技術者向け・確定前の草案）
- トレイ見出し：「読み込み中の Google Drive 参照メモ（N件）」
- 一時性：「これらは参考用のコピーです。MyBrain には保存されません。ページを再読み込みすると消えます。」
- 上限：「AI 相談で使えるのは、上から5件・各メモ約200字までです。」
- AI対象ラベル：「AIで使用」／「AIには渡りません」
- ボタン：「参照を解除」（1件）／「すべて解除」（全部）
- AI通知：「Google Drive参照 N件も参考にします（MyBrainには保存されません）」＋「使うメモ：〇〇／△△ …」

## 5. 変更しないもの（境界）

- **データの持ち方・保存方針**：`driveRefMemos` は引き続き **React state のみ・保存しない**。localStorage・Supabase への永続化はしない（P1〜P6 はすべて**表示の追加**で解く）。
- **Google Drive の挙動・API**：`lib/google/` の読み取り・書き出しロジック、`drive.file` スコープ、トークンの扱いは変更しない。読み取り専用のまま。
- **取り込み（インポート）・双方向同期**：実装しない（「ない」ことを説明するだけ）。
- **AI 送信ロジック**：`buildDriveReferenceBlock`・`askAssistant` の送信内容・送信先・上限値は変更しない。参照0件時の挙動不変も維持。
- **モバイル UI**：一覧・プレビュー・参照・AI 参照はデスクトップのみ。モバイルには広げない（本 MVP の対象外）。
- **本体メモ・検索・フォルダ・お気に入り・一括選択**：参照メモは引き続きこれらに混ぜない（件数・絞り込みの対象外）。
- **Supabase スキーマ・OAuth スコープ・Google カレンダー連携**：変更しない。

## 6. スコープ外（今回やらないこと）

- 参照メモの永続化・自動読み込み・過去分の復元。
- AI に渡す件数／文字数のユーザー設定化（上限値の変更）。
- 参照メモの並べ替え（関連度スコアリング・ベクトル検索・全文インデックス）。
- モバイルへの参照・AI 拡張（OBS32 次フェーズ B として別途）。
- 取り込み（インポート）機能（OBS32 次フェーズ D・要設計）。

## 7. QA チェックリスト草案（実装フェーズで使用・今回は実施しない）

- U1：参照を1件追加すると、検索語なしでもメモ一覧上部にトレイが出て件数が正しい。
- U2：トレイの「参照を解除」で対象1件だけ消え、他は残る。
- U3：トレイの「すべて解除」で全部消え、トレイ自体が消える。
- U4：トレイに一時性の警告（保存されない・再読み込みで消える）が常時表示される。
- U5：6件以上読み込むと、先頭5件に「AIで使用」・6件目以降に「AIには渡りません」が付く。
- U6：AI 通知に、実際に渡る先頭5件のタイトルが表示される。
- U7：参照0件のとき、トレイ・AI通知ともに非表示で、AI 送信内容は従来と完全に同じ（挙動不変）。
- U8：ページ再読み込みで参照メモが消える（保存されていない）。
- U9：既存の Drive 一覧・プレビュー・検索結果ブロック・「参照中」バッジは引き続き動作する。
- U10：モバイル UI は変化しない。
- U11：`npx tsc --noEmit`・`npm run build` が成功する。

## 8. 実装の進め方（提案・別タスク）

- 本 MVP は**表示のみ**なので、`DesktopMemos.tsx` にトレイ表示を1つ足し、AI 通知に先頭5件のタイトル列挙を足す、という小さな2差分に収まる見込み。
- 実装は OBS34 とは別の独立タスクとして、1つずつ小さく行う（スマホ UI を壊さない・`driveRefMemos` の持ち方は変えない）。
- 実装後、上記 U1〜U11 を本番（デスクトップ・ログイン済み・Drive 構成済み）で確認する。
