# モバイル Drive 参照メモ・AI 相談拡張 ― 設計メモ（OBS36）

> **ステータス：調査・設計のみ（未実装）。この OBS36 ではアプリコード・UI・Supabase スキーマ・OAuth スコープ・Google Drive の挙動・API を変更しない。**
> 対象：デスクトップ専用の「Google Drive Markdown 参照メモ」（一覧・プレビュー・参照・AI 参照）を、モバイルへ慎重に広げる（OBS32 次フェーズ B）。
> 前提：`docs/google-drive-markdown-read-search-design.md`（OBS25・デスクトップ全フェーズ完了）／`docs/drive-reference-ux-improvement-design.md`（OBS34〜OBS35R・トレイ UX 完了）／`docs/google-drive-markdown-user-guide.md`（OBS32）。
> 実装対象ファイル（将来）：`app/consult/page.tsx`（モバイル AI 相談）／`lib/ai/consult-ollama.ts`（コンテキスト構築）／新規の小さなモバイル用パネルコンポーネント。

---

## 1. 目的

デスクトップでは「Drive 一覧 → プレビュー → 参照に追加 → トレイで可視化 → AI 相談の参考」まで完成し、本番QAも完了している（OBS35R・U1〜U11 全 Pass）。
モバイルは現在、**書き出し（①）だけ**ができる（メモ保存後のワンタップ書き出し・/history の一括書き出し）。読み返し・参照・AI 参照はできない。
本メモは、モバイルに **最小限の「参照 → AI 相談」体験**を持ち込む MVP を設計する。**スマホ UI を壊さない・小さく・1画面完結**を最優先とする。

## 2. 現状の調査結果

### 2.1 デスクトップ（完成済み・変更しない）
- `components/DriveExportedFilesList.tsx`：一覧（`listDriveMarkdownFiles`）・プレビュー（`readDriveMarkdownFile`＋`markdownToMemo`）・参照に追加。`DriveReferenceMemo` 型もここで定義。
- `components/DesktopMemos.tsx`：`driveRefMemos`（React state・メモリのみ）・参照トレイ（OBS35）・検索参照ブロック・AI 通知・`buildDriveReferenceBlock`（最大5件・各約200字・別ブロックで user メッセージ末尾に追記）。

### 2.2 モバイル（現状）
| 画面 | いまの役割 | Drive 関連 |
|---|---|---|
| `/memos` | メモ作成 | 保存後のワンタップ Drive 書き出しのみ |
| `/history` | メモ一覧・履歴 | 一括エクスポート（ZIP／Google Driveへ書き出し）のみ |
| `/consult` | AI 相談（履歴つきチャット） | なし。`askOllamaConsult` が保存済みメモ・予定を参照（`refTarget`：メモ／予定／両方） |
| `/ai-assist` | AI が参照する情報のトグル設定 | なし |
| `/settings` | 設定（OBS33 でやさしい説明を追加済み） | 説明のみ |

### 2.3 モバイル特有の制約（設計の決め手）
- **ルートをまたぐと state が消える。** デスクトップは `DesktopMemos` 1コンポーネント内で「参照追加→AI相談」が完結するが、モバイルは `/history` と `/consult` が別ページ。参照メモは**メモリのみ・保存禁止**（localStorage・Supabase に入れない）なので、画面をまたぐ設計にすると参照が途中で消えて混乱する。
- → **結論：MVP は `/consult` 1画面にすべてを置く**（読み込みも・トレイも・AI 相談も同じ画面）。

## 3. モバイル MVP 設計

### 3.1 どの画面に置くか（設計判断）
- **`/consult`（AI 相談画面）に一本化する。**
  - 理由①：参照メモの唯一の用途が「AI 相談の参考」なので、使う場所に置くのが最も分かりやすい。
  - 理由②：1画面完結なら「メモリのみ・リロードで消える」制約と自然に一致する（画面を離れたら消える、が直感どおり）。
  - 理由③：`/consult` には既に「AIが参照する情報」セレクターがあり、「参照」という概念の置き場所として一貫する。
- `/history`・`/memos`・`/ai-assist`・設定は**変更しない**（書き出し導線は現状のまま）。新規画面も作らない。

### 3.2 画面構成（`/consult` に追加する1セクション）
「AIが参照する情報」セレクターの近くに、折りたたみ式の **「Google Driveのメモを参考にする」** セクションを追加する（Drive 構成済み・ログイン済みのときのみ表示。既定は閉じる＝既存画面は変わって見えない）。

開いたときの中身（上から順に）：
1. やさしい説明（常時）：
   - 「Google Driveに書き出したメモを、この相談の参考にできます。」
   - 「**この参照メモは一時的です。画面を離れたり再読み込みすると消えます。**」
   - 「**AI相談に使われるのは最大5件までです。**」
2. 「**一覧を確認**」ボタン → タップ時のみ Drive に問い合わせ（`listDriveMarkdownFiles`・自動読み込みなし・OAuth はタップ起点）。
3. ファイル一覧（ファイル名・更新日時）＋各行に「**参照に追加**」（追加済みは「参照中」バッジ）。
4. **読み込み中トレイ**（参照が1件以上のとき）：デスクトップ OBS35 と同じ要素をモバイル向けに縦積みで。
   - 見出し「読み込み中のGoogle Drive参照メモ（N件）」＋「**すべて解除**」
   - 各件：タイトル（無ければファイル名）＋「**AIで使用**」（先頭5件）／「**AIには渡りません**」（6件目以降）＋「**参照を解除**」
5. **送信前通知**（参照が1件以上のとき・質問入力欄の上）：
   - 「Google Drive参照 N件も参考にします（MyBrainには保存されません）」
   - 「使うメモ：〇〇／△△ …」（実際に渡る先頭最大5件のタイトル）

### 3.3 AI への渡し方（デスクトップと同じ規則）
- `lib/ai/consult-ollama.ts` の `askOllamaConsult` に**省略可能な引数**として参照メモ配列を追加する（例：`driveRefs?: DriveReferenceMemo[]`）。
  - **未指定・0件なら送信内容は従来と完全に同じ**（挙動不変。既存呼び出し元は無修正で動く）。
  - 1件以上なら、デスクトップの `buildDriveReferenceBlock` と**同じ上限・同じ形式**（最大5件・本文各約200字・タイトル60字・「Google Drive参照メモ（MyBrain本体ではありません…）」見出しの別ブロック）で user メッセージ末尾に追記し、system に「参照メモは補助・主根拠は保存済みメモ」の1文を足す。
- 上限値はデスクトップ（`DRIVE_REF_AI_MAX_ITEMS=5`／`DRIVE_REF_AI_MAX_CHARS=200`）と**同一とする**（モバイル独自の上限は作らない）。
- ローカルルールエンジン（`buildConsultAnswer`）・参照カード・相談履歴（localStorage）には参照メモを**混ぜない**（履歴に保存すると「保存しない」原則に反するため。AI 回答テキストのみ従来どおり履歴に残る）。

### 3.4 実装方針（将来の実装タスク向け・小さく）
- **再利用するもの（新しい Drive ロジックを作らない）**：`lib/google`（`isGoogleDriveConfigured`・`requestGoogleDriveAccessToken`・`listDriveMarkdownFiles`・`readDriveMarkdownFile`・`DRIVE_MARKDOWN_READ_MAX_BYTES`）／`lib/markdown` の `markdownToMemo`／`DriveReferenceMemo` 型（`components/DriveExportedFilesList.tsx` から **型のみ import**。デスクトップコードは無変更）。
- **新規（小）**：モバイルの見た目（ダーク・ネオン・グラス）に合わせた小さなパネルコンポーネント（例：`components/MobileDriveReferencePanel.tsx`）。`DriveExportedFilesList` はライトテーマのデスクトップ専用スタイルのため**共用しない**（プロップ追加でテーマ分岐するとデスクトップに触れてしまうので避ける）。state（参照メモ配列）は `/consult` ページが持つ。
- **段階分け（それぞれ独立の小タスク）**：
  - **Phase M1**：`/consult` に一覧＋参照追加＋トレイ＋送信前通知＋AI 接続（本設計の MVP）。
  - **Phase M2（任意・後続）**：1件の簡易プレビュー（「内容を確認」）。MVP では省略可 — ファイル名・更新日時で選び、追加後はトレイでタイトルを確認できるため。
  - **検索参照（デスクトップの検索結果ブロック相当）はモバイルに持ち込まない**（当面デスクトップ専用のまま）。

### 3.5 やさしい文言（10〜70代向け・草案）
- セクション見出し：「Google Driveのメモを参考にする」
- 説明：「Google Driveに書き出したメモを、この相談の参考にできます。」
- 一時性：「この参照メモは一時的です。画面を離れたり再読み込みすると消えます。」
- 上限：「AI相談に使われるのは最大5件までです。」
- ボタン：「一覧を確認」「参照に追加」「参照を解除」「すべて解除」
- バッジ：「参照中」「AIで使用」「AIには渡りません」
- 送信前通知：「Google Drive参照 N件も参考にします（MyBrainには保存されません）」＋「使うメモ：…」
- 失敗時：「一覧を取得できませんでした。時間をおいてもう一度お試しください。」（デスクトップと同文）

## 4. 変更しないもの（境界・明文化）

- **MyBrain（Supabase）が正本（source of truth）のまま。** 参照メモは本体メモに一切影響しない。
- **Google Drive Markdown は書き出しコピー・参照用のみ。**
- **取り込み（インポート）はしない。** Drive → MyBrain への保存は作らない。
- **双方向同期はしない。** 書き出しは手動・MyBrain→Drive の一方向のみ。
- **参照メモは保存しない。** React state のみ（メモリのみ）。localStorage・Supabase・相談履歴に入れない。画面を離れる／再読み込みで消える。
- **AI が使うのは読み込んだ参照メモだけ。** 上限はデスクトップと同一（最大5件・本文各約200字）。0件時の送信内容は従来と完全に同じ。
- **デスクトップの挙動は変更しない。**（`DesktopMemos.tsx`・`DriveExportedFilesList.tsx` は無変更。型 import のみ。）
- **OAuth スコープは `drive.file` のまま。** トークンは短命・メモリのみ・保存しない。ユーザー操作起点でのみ Drive に問い合わせる。
- **Supabase スキーマ・Google カレンダー連携・`/history` の一括エクスポート・`/memos` の書き出しボタン・`/ai-assist`・設定画面は変更しない。**

## 5. スコープ外（今回やらないこと）

- モバイルでの検索参照（検索結果ブロック）。
- 参照メモの永続化・自動読み込み・画面またぎの保持（モジュールストア／Context 化）。
- 上限値のユーザー設定化・関連度並べ替え・ベクトル検索。
- 取り込み（インポート）＝ OBS32 次フェーズ D（要設計・別タスク）。
- `/history`・`/memos` への参照 UI の追加（1画面完結を崩すため今回はしない）。
