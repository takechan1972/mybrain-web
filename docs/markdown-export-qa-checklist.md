# Markdown エクスポート 手動QAチェックリスト

MyBrain のメモ Markdown エクスポート（単件ダウンロード／ZIP一括／フォルダ書き出し／Google Drive 手動書き出し）を
リリース前に手動検証するためのチェックリスト。エクスポートは全て手動（ユーザー操作起点）であることの確認を含む。

- 方針：MyBrain（Supabase）が常に本体の保存先（source of truth）。エクスポートは付加的な書き出しで、メモ本体には影響しない。
- 関連：`docs/obsidian-local-save-manual-test.md`（obsidian-local の付加的 Vault 保存の詳細テスト）

---

## テスト環境（前提）

- MyBrain **ログイン済み**であること（未ログイン時の挙動はテスト 1 で確認）。
- ZIP・単件ダウンロード：モバイル／デスクトップの一般的なブラウザで可。
- フォルダ書き出し：デスクトップ **Chrome または Edge**（File System Access API 対応ブラウザ）のみ。
- Google Drive 書き出し：Google Drive 連携の公開設定（環境変数）が構成済みの環境のみ。

---

## テストケース（8 件）

### 1. ログイン必須画面

- **手順**：未ログイン状態で `/history?view=memos`・`/memos`・`/settings`・`/memos/[id]` を開く
- **期待**：いずれもログイン画面へ誘導され、メモ内容・エクスポートUIは表示されない
- **合否**：☑ Pass ☐ Fail（2026-07-08 実施。`/history?view=memos` で確認。下の実施記録を参照）

### 2. モバイル メモ一覧の一括ZIPエクスポート

- **手順**：モバイルで `/history?view=memos` →「一括エクスポート」パネルの「選択」→メモを複数タップ選択→「選択メモを書き出し」→確認ダイアログでOK
- **期待**：`mybrain-memos-YYYY-MM-DD.zip` がダウンロードされ、トースト `N件をZIPで書き出しました` が表示。ZIP内は各メモの `.md`（frontmatter付き）。同名タイトルは `-2.md` の連番で上書きされない
- **備考**：10件以上選択時は追加の確認ダイアログが出る
- **合否**：☐ Pass ☐ Fail

### 3. デスクトップ メモ一覧の一括ZIPエクスポート

- **手順**：デスクトップのメモ一覧（リスト表示）→「一括エクスポート」パネルの「選択してまとめる」→複数選択→「選択メモを書き出し」→確認ダイアログでOK
- **期待**：テスト 2 と同じ ZIP がダウンロードされ、同じトーストが表示される
- **合否**：☐ Pass ☐ Fail

### 4. メモ詳細の単件Markdownダウンロード

- **手順**：`/memos/[id]`（またはデスクトップのメモ詳細）→「Obsidian形式で表示」→「Obsidian用Markdownをダウンロード」／「Obsidian用Markdownをコピー」
- **期待**：`<タイトル>.md` がダウンロードされる（コピーはクリップボードへ）。frontmatter に `id` / `title` / `tags` / `created` / `updated` / `source: mybrain` を含む
- **合否**：☐ Pass ☐ Fail

### 5. フォルダへ書き出し（対応デスクトップブラウザのみ）

- **手順**：デスクトップ Chrome/Edge でメモ一覧→選択モード→複数選択→「フォルダへ書き出し」→フォルダ選択（接続済みVaultがあれば省略）→OK
- **期待**：選んだフォルダの `MyBrain/Memos/` 配下に各メモの `.md` が作成される。既存ファイルは上書きされず `-2.md` の連番になる。トースト `N件をフォルダへ書き出しました`
- **備考**：File System Access API 非対応ブラウザ（モバイル Safari 等）では「フォルダへ書き出し」ボタン自体が表示されない
- **合否**：☑ Pass ☐ Fail（2026-07-08 実施。対応デスクトップブラウザで確認。下の実施記録を参照）

### 6. Google Drive 手動エクスポート（構成済み環境のみ）

- **手順**：Google Drive 連携が構成済みの環境で、メモ一覧→選択モード→複数選択→「Google Driveへ書き出し」→OK→Google の同意ポップアップを許可
- **期待**：Google Drive の `MyBrain/Memos/` 配下に `.md` が作成される。既存ファイルは上書きされず `-2.md` の連番になる。トースト `N件をGoogle Driveへ書き出しました`
- **備考**：未構成の環境では「Google Driveへ書き出し」ボタン自体が表示されない。同意ポップアップがブロックされる場合はブラウザのポップアップ設定を確認
- **合否**：☑ Pass ☐ Fail（2026-07-08 実施。構成済み環境で確認。下の実施記録を参照）

### 7. MyBrain が本体の保存先であること

- **手順**：テスト 2〜6 の各エクスポート後に、アプリのメモ一覧・メモ詳細を再確認する
- **期待**：メモ本体は増減・変化なし（エクスポートで削除・移動されない）。設定画面に「MyBrainが本体の保存先」の説明が表示されている
- **合否**：☑ Pass ☐ Fail（2026-07-08 実施。下の実施記録を参照）

### 8. エクスポートが手動のみであること（自動保存なし）

- **手順**：保存先設定が `mybrain` のままメモを新規保存・更新し、Obsidian Vault と Google Drive を確認する
- **期待**：ボタン操作なしに Vault・Google Drive へファイルが作成されることはない。Google Drive への自動同期は動作しない（保存先 `obsidian-gdrive` でも、保存後に表示されるボタンをタップしたときだけ書き出される）
- **備考**：保存先 `obsidian-local` ＋ Vault 接続済みの場合のみ、保存時に付加的な Vault 書き出しが行われる（詳細は `docs/obsidian-local-save-manual-test.md`）
- **合否**：☑ Pass ☐ Fail（2026-07-08 実施。下の実施記録を参照）

## ブラウザ制限の注意

- **フォルダ書き出し**は File System Access API 対応のデスクトップ Chrome / Edge のみ。モバイルや Safari / Firefox では非対応（ボタン非表示）。
- **モバイル**では Markdown のコピー・単件ダウンロード・ZIP一括ダウンロードが利用できる（フォルダ直接保存は不可）。
- **Google Drive 書き出し**は GIS の同意ポップアップ（ユーザー操作起点）が必要。ポップアップブロックに注意。
- iOS Safari では ZIP や `.md` のダウンロード先が「ファイル」アプリになるなど、端末により保存先の表示が異なる。
