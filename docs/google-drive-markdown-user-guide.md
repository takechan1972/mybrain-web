# Google Drive Markdown 連携 ― かんたんユーザーガイド（OBS32）

> **対象：MyBrain をお使いのみなさま（10〜70代向け・やさしい日本語）。**
> このガイドは、MyBrain と Google Drive の Markdown 連携で「いま何ができるか」「どう考えればよいか」をまとめたものです。
> ドキュメントのみ（アプリの動作説明です）。設計の詳細は `docs/google-drive-markdown-read-search-design.md`、実装状況は `docs/PROJECT_STATUS.md` を参照してください。

---

## 1. いちばん大事なこと（3行まとめ）

- **あなたのメモの「本体（正本）」は、いつも MyBrain の中にあります。**
- **Google Drive の Markdown は「書き出したコピー」「参考用」です。** 本体ではありません。
- **AI 相談で参考にできるのは、あなたがその場で読み込んだ参照メモだけ**です。参照メモは保存されず、画面を閉じると消えます。

---

## 2. いま何ができるか（完成した機能）

MyBrain のデスクトップ画面（パソコン）で、次のことができます。

| できること | かんたんな説明 |
|---|---|
| ① Google Drive へ書き出し | メモを Google Drive の `MyBrain/Memos/` フォルダに Markdown ファイルとして書き出せます（手動・ボタンを押したときだけ）。 |
| ② 書き出し済みの一覧を見る | Google Drive に書き出した Markdown を、ファイル名・更新日時つきで一覧できます。 |
| ③ 中身をプレビューする | 一覧から1件を選ぶと、その内容（タイトル・タグ・本文）を画面で読めます。 |
| ④ 参照に追加して検索する | 読み込んだ Markdown を「Google Drive参照」としてメモ検索の参考にできます（本体メモとは別枠で表示）。 |
| ⑤ AI 相談の参考にする | 参照に追加した Markdown を、デスクトップの AI アシスタント（自由質問）の参考として渡せます。 |

- スマホ（モバイル）でも、メモの一括 Google Drive 書き出し（①）は使えます。
- ②〜⑤（一覧・プレビュー・参照検索・AI 参照）は**パソコンのデスクトップ画面のみ**の機能です。

---

## 3. ユーザーが理解しておくとよいこと（安心のための4点）

### ① MyBrain本体が正本
- メモの正式な保存場所は、いつも MyBrain（あなたのアカウント）です。
- Google Drive へ書き出しても、MyBrain のメモは消えたり移動したりしません。

### ② Google Drive Markdownは書き出しコピー・参照用
- Google Drive の Markdown は、MyBrain から「書き出したコピー」です。
- あとから読み返したり、検索や AI 相談の参考にしたりするための「参照用」です。
- Google Drive の Markdown を編集しても、MyBrain の本体メモには反映されません（取り込み機能はありません）。

### ③ 参照メモは保存されない
- 「参照に追加」した Markdown は、その画面を開いている間だけ有効です。
- ページを再読み込みしたり画面を離れたりすると、参照メモは消えます。
- 参照メモは MyBrain（Supabase）にも、ブラウザ（localStorage）にも保存されません。

### ④ AI相談に使えるのは読み込んだ参照メモのみ
- AI アシスタントが参考にできるのは、あなたがその場で「参照に追加」した Markdown だけです。
- Google Drive にある他のファイルを、AI が勝手に読むことはありません。
- 参照を使うときは、画面に「Google Drive参照 N件も参考にします（MyBrainには保存されません）」と表示されます（黙って混ぜません）。

### すこし詳しい版（ヘルプ用）
- **Google Drive への書き出しについて**
  「メモは MyBrain に保存されています。Google Drive へは、ボタンを押したときだけ Markdown としてコピーが書き出されます。自動保存はされません。」
- **参照・検索について**
  「Google Drive に書き出した Markdown は、一覧・プレビューで読み返せます。『参照に追加』すると、メモ検索の参考にできます（この画面を開いている間だけ・保存はされません）。」
- **AI 相談について**
  「『参照に追加』した Markdown は、デスクトップの AI アシスタントに質問するときの参考にできます。参考にするのは、あなたが読み込んだものだけです。使うときは画面にお知らせが出ます。」

### やってはいけない誤解（FAQ 風）
- Q：Google Drive の Markdown を書き換えたら MyBrain も変わりますか？ → A：変わりません。Google Drive は書き出したコピーです。
- Q：参照メモはずっと残りますか？ → A：残りません。画面を閉じると消えます。もう一度使うときは、また「参照に追加」してください。
- Q：AI は Google Drive の全部を読みますか？ → A：読みません。あなたが「参照に追加」したものだけです。

## 5. いまの制限（できないこと）

- **他アプリが置いたファイルは見えません。** MyBrain 自身が書き出した Markdown だけが一覧・参照できます（Google の `drive.file` の仕様。Obsidian などが直接置いたファイルは対象外）。
- **取り込み（インポート）はできません。** Google Drive → MyBrain への保存・双方向同期はありません。
- **参照メモは保存されません。** 画面を閉じると消えます（自動読み込み・過去分の復元もありません）。
- **AI 参照はデスクトップの自由質問のみ。** モバイルの AI 相談（consult）や、要約・整理には参照は渡りません。
- **AI に渡せる参照の量には上限があります。** 最大5件・1件あたり本文は約200字までです。
- **AI 参照は「関連度の並べ替え」をしません。** 読み込んだ順の先頭から使います（ベクトル検索・全文インデックスはありません）。

## 6. 次にできること（おすすめの次フェーズ・案）

> いずれも**未着手のアイデア**です。やる場合はそれぞれ独立した小タスクとして、スマホ UI を壊さないよう慎重に進めます。

- **A. ヘルプ／設定への文言反映（UI 変更・小）**：本ガイド4章の文言を、設定画面やヘルプに載せる。まずデスクトップから。
- **B. モバイルへの参照・AI 拡張（中）**：デスクトップのみの一覧・プレビュー・参照・AI 参照を、モバイルにも慎重に広げる。
- **C. 参照の一時保持（メモリ内の使い勝手改善・小〜中）**：参照メモの並べ替えや、AI に渡す件数の選択などの小さな操作性向上（保存はしないまま）。
- **D. 取り込み（インポート）機能の検討（大・要設計）**：Google Drive → MyBrain への取り込みは影響が大きいので、別途しっかり設計してから判断する。
- **E. 検索の高度化（大・将来）**：ベクトル検索・埋め込み・関連度スコアリングは、別設計の将来タスクとして扱う。

## 付記：技術的な前提（変更していないこと）

- MyBrain（Supabase）が source of truth。Google Drive Markdown は付加的なエクスポートコピー・参照用。
- OAuth スコープは `drive.file` のまま（変更なし）。追加同意は不要。
- Google Drive への書き込み・変更・削除はしません（読み取り・書き出しのみ、いずれも手動）。
- アクセストークンは短命・メモリのみ・保存しません。
- Supabase スキーマ・Google カレンダー連携は変更していません。
