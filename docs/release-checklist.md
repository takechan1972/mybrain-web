# AIプラ スマホ版 実機テスト チェックリスト（販売前）

対象：EAS `preview` プロファイルでビルドした実機テストビルド（Android APK / iOS internal）。
本書はストア公開前の「実機での動作確認」用です。

---

## 0. ビルド前の前提確認
- [ ] `eas.json` の `preview.env.EXPO_PUBLIC_API_BASE_URL` を**実在するテスト用APIのURL**に置き換えた
- [ ] テスト用バックエンド（ai-iphone-backend）が起動・公開されており、端末からアクセス可能
- [ ] バックエンド `.env` に `TERMS_URL` / `PRIVACY_URL` / `TOKUSHOHO_URL` を設定（未設定なら「準備中」表示になる）
- [ ] 環境変数を変更した場合は**再ビルドが必要**（EXPO_PUBLIC_* はビルド時に埋め込まれる）

## 1. 起動・外観
- [ ] 初回起動できる（クラッシュしない）
- [ ] スプラッシュがブランド青(#208AEF)＋アイコンで表示される（Expo既定ロゴが出ない）
- [ ] ホーム画面のアプリアイコンが「AIプラ」アイコンで表示される
- [ ] アプリ名が「AIプラ」で表示される
- [ ] React/Expo のテンプレ画像・文言が画面に出ない

## 2. 設定画面
- [ ] 設定画面が開ける
- [ ] 「はじめての使い方」が表示される
- [ ] 現在のプラン・契約状態が表示される
- [ ] お問い合わせ・解約申請・規約リンクが上部に見える
- [ ] 「詳細設定（上級者向け）」は初期状態で閉じている
- [ ] 通常画面に localhost / mock / debug 等の開発者向け文言が出ない

## 3. 接続状態チェック
- [ ] 「接続を確認」で `/health` を確認できる
- [ ] 接続成功時「接続状態：確認済み」
- [ ] 接続不可時「接続状態：確認できません」（アプリが落ちない）
- [ ] URLや内部情報が画面に出ない

## 4. AIチャット
- [ ] メッセージを送信できる
- [ ] 送信中「AIが考え中…」のローディングが出る
- [ ] 応答が表示される
- [ ] 通信失敗時にやさしいエラー文が出る（内部詳細は出ない）
- [ ] 利用上限時「本日の利用上限に達しました。…プラン変更をご検討ください。」が出る

## 5. メモ管理
- [ ] メモを新規作成できる
- [ ] 保存後に内容が残る
- [ ] 編集できる
- [ ] 削除時に確認ダイアログが出る → 削除できる
- [ ] タグ・検索・ピン留めが機能する
- [ ] 空状態の案内が分かりやすい

## 6. 予定管理
- [ ] 予定を新規作成できる
- [ ] 日付フィルター（今日/明日/今週）が機能する
- [ ] 編集できる
- [ ] 削除時に確認ダイアログが出る → 削除できる
- [ ] 空状態の案内が分かりやすい

## 7. データ保持
- [ ] アプリを完全終了して再起動してもメモが残る
- [ ] 予定が残る
- [ ] AIチャット履歴が残る

## 8. 契約・サポート
- [ ] お問い合わせ画面に遷移できる
- [ ] 必須未入力でエラーになる（メール/件名/内容10文字以上）
- [ ] お問い合わせを送信でき、受付番号・送信日時が表示される
- [ ] 解約申請画面に遷移できる
- [ ] 確認チェックなしでは送信できない
- [ ] 解約申請を送信でき、受付番号が表示される
- [ ] 解約申請しても即時にアカウント・データが消えない
- [ ] 規約・プライバシー・特商法リンクが開く（未設定時は「準備中」）

## 9. レート制限・エラー文言
- [ ] お問い合わせ/解約を短時間に連続送信すると「短時間に送信が集中しています。…」が出る
- [ ] 通信失敗時「通信できませんでした。…」が出る
- [ ] サーバー停止時「サーバーに接続できませんでした。…」が出る

## 10. production 接続仕様
- [ ] `EXPO_PUBLIC_APP_ENV=production` かつ `EXPO_PUBLIC_API_BASE_URL` 未設定のビルドで、localhost へ接続しない
- [ ] 上記の場合「アプリの接続設定が完了していません。販売元へお問い合わせください。」が出る
- [ ] `EXPO_PUBLIC_API_BASE_URL` 設定時はそのURLへ接続する

## 11. セキュリティ
- [ ] APIキー・token・password・Bearer が画面に出ない
- [ ] 問い合わせ本文・解約理由の全文がログに出ない
- [ ] エラーの内部詳細（スタックトレース等）が画面に出ない

---

## バージョン管理メモ（アップデート時に上げる）
- iOS: `app.json > expo.ios.buildNumber`（ビルドごとに +1）
- Android: `app.json > expo.android.versionCode`（ビルドごとに +1）
- 表示バージョン: `app.json > expo.version`（機能リリースごとに 1.0.1 等）
- ※ `eas.json` の production は `autoIncrement: true`（ビルド番号自動加算）
