# MyBrain Development Rules

- Work one small task at a time.
- Do not make large changes at once.
- Do not break the existing smartphone UI.
- Run npx tsc --noEmit after code changes.
- Run npm run build after code changes.
- Do not change Supabase schema unless explicitly instructed.
- chatbot_knowledge is admin-managed global Q&A knowledge.
- Do not add user_id to chatbot_knowledge.
- contact_inquiries is for inquiry storage.
- chatbot_knowledge is for FAQ/Q&A storage.
- Smartphone UI has priority.
- Use simple Japanese labels.
